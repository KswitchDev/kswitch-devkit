# Release Legal Checklist

Complete this checklist before publishing a Developer Edition release, release
archive, package, image, or website update.

- [ ] Licence map reviewed: `LICENSE.md`
- [ ] Developer Edition Licence reviewed by counsel:
      `LICENSES/KSWITCH-DEVELOPER-EDITION-LICENSE.md`
- [ ] Commercial-use wording reviewed: `COMMERCIAL-USE.md`
- [ ] Trademark wording reviewed: `TRADEMARKS.md`
- [ ] Privacy/telemetry posture reviewed: `PRIVACY.md`
- [ ] Security reporting posture reviewed: `SECURITY.md`
- [ ] Contribution policy reviewed: `CONTRIBUTING.md`
- [ ] Dependency licence policy reviewed: `DEPENDENCY-LICENSE-POLICY.md`
- [ ] Third-party notices refreshed: `THIRD-PARTY-NOTICES.md`
- [ ] SDK package metadata points to Apache-2.0 where intended
- [ ] Developer Edition runtime/devkit surfaces point to the Developer Edition
      Licence
- [ ] `scripts/validate_legal_boundary.py` passes
- [ ] Website copy says source-available, non-commercial, not open source, and
      commercial use requires a separate agreement
- [ ] Public docs avoid unapproved "secure", "bank-grade", "compliant",
      "production ready", "enterprise ready", "zero trust", or regulated-use
      claims for Developer Edition
- [ ] Issue and PR templates warn against secrets, customer data, production
      logs, personal data, private URLs, and proprietary third-party code
- [ ] Export/sanctions and jurisdiction-specific terms reviewed
