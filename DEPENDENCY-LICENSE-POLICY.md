# Dependency Licence Policy

This policy applies to npm, PyPI, Go modules, generated clients, vendored code,
documentation assets, examples, container image layers, and bundled runtime
artefacts.

## Generally Approved

- Apache-2.0
- MIT
- BSD-2-Clause
- BSD-3-Clause
- ISC
- CC0 for data/examples where attribution is not required

## Requires Review Before Shipping

- MPL, EPL, CDDL, LGPL, GPL, AGPL
- SSPL, BUSL, Commons Clause, Elastic License, Redis Source Available License
- PolyForm licences in dependencies
- custom EULAs or non-standard source-available licences
- unknown, missing, or ambiguous licences
- generated code with unclear source terms
- container image layers with source-offer, copyleft, or redistribution
  obligations

## Blocked Without Written Approval

- no-licence third-party code;
- dependencies that require relicensing KSwitch SDKs, Developer Edition, or
  runtime materials;
- dependencies whose terms permit only non-commercial use in Apache-2.0 SDK
  packages;
- dependencies that prohibit security analysis, benchmarking required by
  release gates, or redistribution needed by DevKit.
