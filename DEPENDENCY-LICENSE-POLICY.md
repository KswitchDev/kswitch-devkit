# Dependency Licence Policy

Public release requires dependency licence review for Python, npm, Go, devkit
source, generated clients, vendored code, documentation assets, examples,
scripts, OS packages, and container image layers.

## Allowed Without Additional Review

- Apache-2.0
- MIT
- BSD-2-Clause
- BSD-3-Clause
- ISC

## Counsel/Product Review Required

- MPL
- EPL
- CDDL
- LGPL
- GPL
- custom EULAs
- licences with source-offer obligations
- any dependency with unclear or missing metadata

## Blocked Unless Explicitly Approved

- AGPL
- SSPL
- BUSL
- Commons Clause
- Polyform non-commercial/network variants
- no-licence code
- unknown licences

Release validation must fail on unknown, blocked, incompatible, or unreviewed
licences unless an approved exception is recorded in the EP-227 evidence.
